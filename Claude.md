# CLAUDE.md — Metaball Morph Studio(ALIAS 点阵形变动画工具)

> 本文件是 Claude Code 的项目说明书。开始任何任务前先完整读一遍。
> 参考实现在 `legacy/metaball-morph-studio-v4.html`(单文件,可直接在浏览器打开运行)。

---

## 1. 这个项目是什么

为自动驾驶汽车的车尾 LED 点阵灯语(eHMI)建造专属创作工具:
设计师画形状/打文字 → 自动填充成点阵 → 多状态序列间平滑形变(metaball morph)→
导出 PNG 序列/视频,经 LedArt 下发到 HD-C16L 控制器驱动的 P2.5 LED 模组上屏。

类比:视频有 After Effects、界面有 Figma,这是"点阵形变"这一媒介的第一个编辑器。
它同时是研究仪器:产出物用于 HRI 用户测试与 8/22 画廊展览(Woods-Gerry)。

## 2. 美学纲领(所有动效决策的最高准则)

**"光不闪烁,光生长。"(Light doesn't flash; it grows.)**

- 状态间永远连续形变(morph),禁止硬切换、禁止闪烁。
- 一切运动用缓动(默认 smootherstep),速度与加速度在端点连续。
- 停留状态带低频呼吸漂移(0.1–0.5 Hz),幅度极小。
- 时序错峰(stagger)制造"波浪式生长",而非全体齐动。
- 任何新功能若引入 3–30 Hz 的亮度振荡,属于光敏安全红线,必须警告或禁止。

## 3. 现状(v4 已实现,迁移时必须保持功能等价)

单文件约 1100 行,分层架构:

- **输入层**:矢量形状对象模型(rect / ellipse / text),布尔加减,选择/移动/四角缩放,
  单点手动补点;形状列表 → 光栅化蒙版 → 采样成点。
- **采样层**:4 种算法 — 方格网格、六角网格、泊松盘(dart-throwing + 网格加速)、仅轮廓。
  单状态点数上限 1500(超出自动抽稀)。
- **引擎层**:纯函数、无状态 — `sampleFrame(g)` 给定全局时间返回 {balls, color}。
  点配对 4 种策略(sortXY / angle / greedy / random);lerp + easing + stagger + 双正弦漂移。
- **序列层**:states[](每状态含 shapes、manual、color、hold、dur),
  segments = [停留, 过渡, …, 可选尾→首无缝过渡]。
- **渲染层**:CPU 逐像素场函数 f(p)=Σr²/d²,阈值+柔度出软边,gamma 校正,
  **tile 分块加速**(24px 格,球按影响半径 6r 登记入格)。
- **导出**:PNG 序列(JSZip,离线逐帧确定性渲染,任意分辨率,拉伸/等比两种适配,
  上限 1200 帧)+ WebM 实时录制(MediaRecorder)。
- **工程**:保存/打开 .json(v4 格式,向后兼容 v3 的 A/B 格式);
  撤销/重做(Ctrl+Z / Ctrl+Shift+Z,60 步,快照制)。
- **UI**:PS 式布局(左工具条 V/R/E/T/D,右属性栏,底部时间轴,顶部状态胶片条 + 缩略图),
  洋葱皮,轨迹显示,画幅参考线,中文界面。

## 5. 架构铁律(违反 = 错误)

- 引擎保持**纯函数**:任意时刻 g 可凭空求值。这是时间轴可拖、导出确定性、
  预览=导出一致性的根基。预览与导出必须共用 `sampleFrame`。
- 数据层(JSON 可序列化)与画布对象(canvas/ctx)严格分离;快照/存档只碰前者。
- 工程文件格式变更必须向后兼容(能读 v3 A/B 与 v4 states)。
- 渲染性能红线:480×280 预览 + 1500 点必须 ≥ 30fps(保住 tile 分块)。
- 不加服务器、不加登录、不加数据库;一切本地。localStorage 仅用于自动保存。

## 6. 关键公式速查(实现新采样/动效时用)

- lerp:`a+(b−a)t`;smoothstep:`3t²−2t³`;smootherstep:`6t⁵−15t⁴+10t³`
- 场函数:`f(p)=Σ rᵢ²/|p−cᵢ|²`,f≥thr 实心;软边 = smoothstep(thr−soft, thr+soft, f)
- 半调:`r = R_max·√B`(亮度开根号,因感知墨量∝面积)
- **融合判据(实体⇄点云)**:间距 s 的点阵成实体 ⟺ `r ≥ (s/2)·√(thr/2)`。
  "实体分解"动效 = 把 dotR(或 thr)做成随时间缓动的通道,穿过该临界线。
- **Vogel 向日葵螺旋(结构化实心填充)**:第 n 点 `r=c·√n, θ=n·137.508°`(黄金角)。
  确定性、无行列感、有中心韵律 — 用于替代"太随机"的泊松盘。
- 同心环采样:第 k 环半径 k·s,放 round(2πk) 个点。
- 泊松盘:飞镖投掷 + cell=minD/√2 网格加速,邻域查 5×5。
- gamma:输出 `a^γ`,LED 建议 γ≈2.2 起步,真机对比微调。

## 7. 硬件与导出约束

- 目标设备:HD-C16L 异步控制器 + P2.5 模组(单模组 128×64 px),LedArt 下发。
- 分辨率预设:480×280(画布)/ 768×64(6 模组横排)/ 384×128(6 模组 3×2)/
  2560×64(20 模组横排)/ 1920×1080(展厅高清)。
- LED 版永远不需要 GPU;展厅实时高密度版才考虑 WebGL/shader,
  且只重写渲染层,引擎与数据层不动。
- ffmpeg 合成参考:`ffmpeg -framerate 30 -i frame_%04d.png -pix_fmt yuv420p out.mp4`

## 9. 代码风格

- Vanilla JS(ES modules),无框架;注释用中文,解释"为什么"而非"是什么"。
- 命名沿用现有词汇:state / shape / dots / pairs / segs / sampleFrame / renderField。
- 采样器、匹配策略保持纯函数并可独立单测(node 断言:融合判据、Vogel 均匀性、序列时长求和)。
- UI 文案中文;快捷键标注在按钮 title。

## 10. 验证方式

- `node --check` 每个模块;`npm test` 跑纯函数断言。
- 手动清单:三状态示例(待机圆→GO→警示条)迁移前后行为一致;
  导出 384×128 @30fps 无缝循环,首尾帧 diff 应为呼吸噪声级差异。
- 性能:1500 点预览帧率 console 打印验证 ≥30fps。
